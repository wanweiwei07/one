# ヒューマノイド（Linx L1）

ヒューマノイドはアームと同じ `MechBase` です。複数のチェーンを登録するだけです。
Linx **L1** の上半身は `'left_arm'`、`'right_arm'`、`'left_arm_waist'`、
`'right_arm_waist'`、`'neck'` を登録します。2つのクラスが提供されています:

- `L1` — 素の本体（アームの先端は `*_arm_link_6` フランジで終わります。自前の
  エンドエフェクタを取り付けてください）。
- `L1O6` — フランジに2つの Linkerbot **O6** 多指ハンドをあらかじめ取り付けた `L1`。

ハンドは取り付けられた子であるため、本体チェーンを下りハンドの tcp まで
**クロスオブジェクト IK** を解いて対象に到達します:

```python
robot.ik(pos, rotmat, chain='left_arm', tcp=robot.left_hand.tcp('power_center'))
```

`'left_arm_waist'` チェーンは腰関節を加え、胴体がリーチを補助できるようにします
（人のように、胴体を静止させてアームに作業をさせるには、最近傍解メトリックで
これに重みを付けます）。

## 計画された把持によるピンチアンドプレース

この例では L1O6 の左手が **対向把持**（手作業で選んだ姿勢ではない）を用いて小さな
バニーを拾います。多指ハンドは `spawn_jaw('pinch')` を介して平行ジョープランナに
自身を提示し、これが較正済みのジョービューを返します。各把持はピックアンドプレースの
キーフレームになり、自由空間のホップは RRT で計画されます:

```python
--8<-- "examples/test_l1o6_pinch_bunny.py"
```

注目すべき点:

- `hand.spawn_jaw('pinch')` は多指ハンドを `antipodal` が期待するインタフェースへ
  適合させます。返される各把持はワールドのピンチ中心姿勢 + 幅です。
- 動くのは `'left_arm_waist'` チェーンのみで、本体の残りは固定されます。
- ウィンドウなしで計画/IK の検証を実行するには `ONE_HEADLESS=1` を設定します。

## スケルトンの可視化

`L1` 自身の `__main__` は本体を半透明（`robot.alpha = 0.3`）にレンダリングし、
`KineVisualizer` のスケルトンをオーバーレイします。これはリンクごとのワールド変換を
読み取るため、*分岐した*ヒューマノイドツリーを正しく描画します（両腕は直列につながれ
ているのではなく、腰から分岐します）。[運動学](kinematics.md#visualizing-the-mechanism)
を参照してください。
